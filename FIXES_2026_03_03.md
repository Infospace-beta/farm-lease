# Issues Fixed - March 3, 2026

## Issue 1: Owner Account Had Admin Privileges ✅ FIXED

### Problem
The account `owner@gmail.com` was incorrectly set with:
- `role = 'admin'` (should be `'landowner'`)
- `is_staff = True` (should be `False`)

This caused the owner to see the admin dashboard instead of the owner dashboard when logging in.

### Solution
1. **Fixed the specific account**: Changed `owner@gmail.com` from `role='admin'` to `role='landowner'`
2. **Added safeguard**: Updated the `User` model's `save()` method to automatically enforce:
   - Only users with `role='admin'` can have `is_staff=True`
   - All other roles (farmer, landowner, dealer) automatically get `is_staff=False` and `is_superuser=False`

### Files Modified
- [backend/farmlease/accounts/models.py](backend/farmlease/accounts/models.py) - Added `save()` method

### Verification
Run `py fix_owner_permissions.py` to verify all users have correct permissions:
```
landowner    | owner@gmail.com                | regular  ✓
```

When logging in with `owner@gmail.com`, you will now be redirected to `/owner` (owner dashboard), not `/admin`.

## Issue 2: Signup Returns 400 Bad Request ✅ FIXED

### Problem
The signup endpoint was returning a 400 error because the `SignupSerializer.create()` method wasn't properly extracting the password before calling `User.objects.create_user()`.

### Root Cause
The original code passed the password inside `**validated_data`, which caused issues with Django's `create_user()` method. The password needs to be extracted and passed as a named parameter.

### Solution
Updated the `create()` method in `SignupSerializer` to:
1. Extract the `password` from `validated_data`
2. Pass it explicitly to `create_user(password=password, **validated_data)`

### Files Modified
- [backend/farmlease/accounts/serializers.py](backend/farmlease/accounts/serializers.py) - Fixed `SignupSerializer.create()` method

### Testing
Tested with curl and successfully created a user:
```bash

  -H "Content-Type: application/json" \

    "first_name": "Test",
```
